# 分析方法の定義

分析をすすめるときのプロセスについて。

```
・目的の明確化（ex. 売上高の増加）
・仮説の洗い出し（ex. 客数の減少, 客単価の減少）
・分析方法の定義（ex. 顧客別の売上推移, 顧客別の客数推移）
・情報（データ）の収集
・具体的な分析

【数字を味方につける：初級編】ビジネスの現場で使えるデータ分析 | Udemy
```


```
ステップ1 目的を認識し、メインの論点（イシュー）を設定する
ステップ2 いったん（なるべく最終的な）仮説を持つ
ステップ3 仮説に従い、どのような情報や分析が必要なのかの当たりをつける
ステップ4 実際に情報を収集する
ステップ5 分析を行う
ステップ6 分析結果を効果的に伝える工夫をする

グロービズ（著）嶋田毅（執筆）正しい意思決定のための「分析」の基礎技術 PHP ビジネス新書 2018
```


```
1. 目的設定
2. 分析計画
3. データ設計
4. データ収集・保存
5. データの前処理
6. 分析手法選択と適用
7. 分析結果の解釈
8. 施策の提案
9. 実施と検証
10. 反省、さらなる改善のために...

あんちべ（著）データ解析の実務プロセス入門 森北出版 2015
```


```
1. 分析要件の確認
    1. 分析結果はどのようにビジネスに活用されるのか
    2. 何が対象か
    3. どんな出力値が必要か
    4. モデルにはどの程度解釈性が必要か
    5. 分析結果をどのように評価するか
    6. 利用する（できる）データに成約はあるか
    7. 処理時間に制約はあるか
    8. 環境面の制約はあるか
2. アプローチ概要の決定
3. データ理解
4. 分析設計
5. 分析実施
6. 結果考察・改善方針検討
```
